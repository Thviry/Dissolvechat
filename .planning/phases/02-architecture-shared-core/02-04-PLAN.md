<objective>
Fix the UAT gap from Phase 02: all protocol signing calls receive undefined instead of the auth private key because useIdentity exposes authPrivKey (CryptoKey) but callers destructure authPrivJwk.

Purpose: The app is currently broken on login and account creation. signObject receives undefined, falls through the CryptoKey guard, and attempts importKey("jwk", undefined) — which throws "required JWK member kty was missing". This blocks every authenticated relay operation.

Output: Both useIdentity hooks expose authPrivJwk as an alias for authPrivKey in their return objects. Both App.jsx files use identity.authPrivKey in handleBlockPeer. All signing paths receive the CryptoKey that signObject already handles.
</objective>
