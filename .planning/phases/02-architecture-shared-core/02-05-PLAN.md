<objective>
Fix two separate bugs that prevent messages from being sent or received:

(A) sendEnvelope returns a raw Error object when all relay fetches are rejected (network unreachable). useMessaging expects a Response-like object with .ok/.status/.json() — the Error object causes resp.ok to be undefined, resp.json() to throw, and the logged error to appear as "undefined".

(B) useIdentity exposes e2eePrivKey (CryptoKey) but not an e2eePrivJwk alias. useMessaging destructures e2eePrivJwk and gets undefined. e2eeDecrypt then calls importEcdhPrivateKey(undefined) which throws. The silent catch means all received messages fail to decrypt.

Purpose: Close UAT gap — "Messages can be sent and received correctly" (Test 3, severity: major)
Output: Working send and receive with a relay running. Correct error reporting (503, not "undefined") when relay is down.
</objective>
